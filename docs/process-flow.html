Local-first orchestration map

One coordinator. Fresh agents. Durable handoffs.

CDO is installed once for the user, then initialized inside each target repository. Every role receives a durable assignment, reports back through lifecycle hooks, and advances only after evidence reconciliation.

User-level install Plugin, MCP server, hooks, and the `cdo` command are registered for the current machine user.
Project init Each Git repo receives `.codex/workflow.toml`, role agents, tracked workflow files, and ignored runtime state.
Accountable handoff The coordinator assigns an owner, waits for completion, validates the artifact and Git evidence, then routes the next role.

Install Boundary

The system has a machine-level plugin layer and a repo-local workflow layer. They are intentionally separate.

Delivery Lifecycle

Select a stage to pin its gate. The sequence is built from the README's end-to-end flow and state machine.

draft Human objective

The user describes the desired change in one Codex terminal. The coordinator starts or resumes durable state.

awaiting_plan_approval Planner artifact

A planning assignment binds the planner to an exact input, output, operation key, and artifact contract.

approved Human approval

The exact persisted plan is approved. CDO stores approver, timestamp, and SHA-256.

executing Executor writes

Executor works in the phase worktree after assignment start and exclusive writer-lease acquisition.

reviewing Independent review

Reviewer checks the task or whole phase from source evidence and returns GO, GO WITH FIXES, or NO-GO.

remediating Fixer loop

Fixer remediates accepted findings, commits evidence, and sends the work back for fresh review.

complete Browser and publish gate

Customer-visible UI needs live browser proof. Merge or deployment still requires explicit human instruction.

queued 1. Assign owner

Coordinator records role, stage, operation, paths, expected artifact, commit range, and attempt.

bind start 2. Acknowledge

The coordinator binds the returned child ID explicitly; the start hook does it automatically when unambiguous.

running 3. Do the work

The coordinator keeps the turn alive while the assigned specialist produces its evidence.

bind stop 4. Notify lead

The stop hook notifies the lead; explicit binding recovers missed or ambiguous lifecycle events.

nextAction 5. Verify and route

Artifact, Git, lease, and state checks decide reviewer, fixer, browser QA, retry, block, or completion.

A stopped agent is not a successful agent. Missing or malformed evidence receives one fresh-session retry; the second failure blocks for human reconciliation.

Agent Roles

Planner and reviewer spend the higher reasoning budget. Worker-style roles use the medium Terra default.

Coordinator

gpt-5.6-terramedium

Assigns owners, waits for results, reconciles evidence, and immediately routes the next accountable role.

Planner

gpt-5.6-solhigh

Reads the repo and writes complete implementation contracts without mutating source.

Executor

gpt-5.6-terramedium

Implements approved task briefs after acquiring the writer lease.

Reviewer

gpt-5.6-solhigh

Independently reviews exact ranges and whole phases before completion.

Fixer

gpt-5.6-terramedium

Fixes accepted review findings and maps every fix to evidence.

Browser verifier

gpt-5.6-terramedium

Checks real UI workflows at desktop and mobile sizes and writes browser proof.

State And Files

The handoff contract is tracked. Runtime bookkeeping stays local and ignored.

Plugin source

  • agents/*.toml native role templates
  • src/*.ts CLI, MCP, state, gates, leases
  • hooks/hooks.json guardrail hook definitions

Tracked per project

  • .codex/workflow.toml model, effort, git, browser, and credential policy
  • .codex/cdo-managed.json installed template version and hashes
  • .codex/agents/*.toml copied role templates for the repo
  • .codex/workflows/* spec, plan, tasks, reports, reviews, browser artifacts

Ignored runtime

  • .codex/workflow-runtime/state.json current workflow status
  • .codex/workflow-runtime/sessions.json assignment and handoff ledger
  • .codex/workflow-runtime/events.jsonl append-only event journal
  • Writer lease and retry counters for the active workflow

Important boundary: CDO can create local commits through executor and fixer agents, but it does not merge, deploy, or mutate production without a current explicit human instruction.

Resume model: a fresh Codex session reconciles any stopped assignment, then reads Git, tracked Markdown, and runtime state instead of trusting old chat history.
